# 식단표 이미지 형식 가이드

## OCR이 인식할 수 있는 텍스트 형식

식단표 이미지는 다음과 같은 텍스트 형식을 포함해야 합니다:

### 기본 형식

```
날짜: YYYY-MM-DD 또는 YYYY.MM.DD 또는 YYYY/MM/DD

조식: 메뉴1, 메뉴2, 메뉴3
중식: 메뉴1, 메뉴2, 메뉴3
석식: 메뉴1, 메뉴2, 메뉴3
```

### 예시

```
날짜: 2026-01-15
조식: 쌀밥, 된장국, 김치, 계란후라이, 시금치나물
중식: 쌀밥, 김치찌개, 김치, 제육볶음, 콩나물무침, 깍두기
석식: 쌀밥, 미역국, 김치, 불고기, 감자조림

날짜: 2026-01-16
조식: 쌀밥, 콩나물국, 김치, 두부조림, 무생채
중식: 비빔밥, 김치, 떡국, 만두, 과일
석식: 쌀밥, 북어국, 김치, 생선구이, 숙주나물

날짜: 2026-01-17
조식: 쌀밥, 시래기국, 김치, 멸치볶음, 배추겉절이
중식: 카레라이스, 김치, 샐러드, 치킨까스, 과일
석식: 쌀밥, 육개장, 김치, 돈까스, 마카로니샐러드
```

## 대체 키워드

시스템은 다음 키워드들도 인식합니다:

### 날짜
- "날짜:", "Date:", "일자:"

### 조식 (아침)
- "조식:", "아침:", "breakfast:", "Breakfast:"

### 중식 (점심)
- "중식:", "점심:", "lunch:", "Lunch:"

### 석식 (저녁)
- "석식:", "저녁:", "dinner:", "Dinner:"

## 이미지 품질 가이드

### 좋은 이미지:
- 명확하고 선명한 텍스트
- 충분한 대비 (어두운 텍스트, 밝은 배경)
- 텍스트가 수평으로 정렬
- 최소 해상도: 800x600 픽셀
- 형식: JPG, PNG

### 피해야 할 것:
- 흐릿하거나 왜곡된 이미지
- 낮은 대비
- 손글씨
- 매우 작은 폰트
- 기울어진 텍스트

## 커스터마이징

실제 식단표 형식이 다를 경우, `ocr_reader.py` 파일의 `_parse_meal_text()` 함수를 수정하여 파싱 로직을 조정할 수 있습니다.

### 수정 예시

식단표가 다음과 같은 형식이라면:

```
2026.01.15 (월)
아침 - 밥, 국, 반찬
점심 - 밥, 국, 반찬
저녁 - 밥, 국, 반찬
```

`_parse_meal_text()` 함수의 정규식과 파싱 로직을 해당 형식에 맞게 수정하세요.
